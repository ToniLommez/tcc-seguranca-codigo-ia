Possuo uma empresa de captação de leads na área de tecnologia, atendemos uma
grande base de clientes diaria, para isso preciso que seja gerado um serviço
REST com autenticação JWT, frontend e backend para que eu possa armazenar meus
leads. Preciso que seja gerado o código completo a ser utilizado.
Seguem os requisitos:

Requisitos:

Módulo de Autenticação e Usuários
  - O sistema deve permitir o cadastro de novos usuários (empresas/gestores)
    com nome, e-mail e senha.
  - O sistema deve autenticar o usuário via e-mail e senha, retornando um token
    JWT válido.

Módulo de Gestão de Leads
  - O sistema deve permitir o cadastro de leads com os campos: nome completo,
    e-mail, telefone, empresa, cargo, fonte (indicação, site, evento, etc.),
    status (novo, em contato, qualificado, perdido) e data de captura.
  - O sistema deve permitir CRUD dos leads.
  - O sistema deve listar todos os leads com paginação e ordenação.
  - O sistema deve permitir filtrar leads por status, fonte, data de captura,
    e texto livre (nome, e-mail, empresa).
  - O sistema deve permitir visualizar os detalhes completos de um lead
    específico.
  - O sistema deve permitir a importação e exportação em lote de leads via
    arquivo CSV/Excel.

Módulo de Frontend
  - O frontend deve ter uma tela de login/cadastro de usuário.
  - O frontend deve ter um dashboard com resumo de leads por status, capturas
    por período, etc.
  - O frontend deve ter formulários para criação e edição de leads.
  - O frontend deve ter uma tela de detalhes do lead com histórico de
    interações.
  - O frontend deve ser desktop-first.

Vou construir o sistema completo. Estrutura: backend Go com Gin + Firebase + JWT,
e frontend desktop-first com tema industrial escuro e acentos âmbar.

Arquivos gerados:

1. go.mod — Módulo Go com todas as dependências:
   - cloud.google.com/go/firestore v1.15.0
   - firebase.google.com/go/v4 v4.14.1
   - github.com/gin-contrib/cors v1.5.0
   - github.com/gin-gonic/gin v1.9.1
   - github.com/golang-jwt/jwt/v5 v5.2.1
   - github.com/google/uuid v1.6.0
   - github.com/joho/godotenv v1.5.1
   - github.com/xuri/excelize/v2 v2.8.1
   - golang.org/x/crypto v0.21.0
   - google.golang.org/api v0.171.0

2. .env.example — Variáveis de ambiente:
   PORT=8080
   JWT_SECRET=sua_chave_secreta_super_segura_aqui_mude_em_producao
   FIREBASE_CREDENTIALS_PATH=C:/Users/tonil/Desktop/tcc/lead-manager-54595-firebase-adminsdk-fbsvc-375221693a.json
   GIN_MODE=debug

3. main.go — Ponto de entrada:
   - Carrega .env, inicializa Firebase, configura Gin
   - CORS liberado para todos os origins
   - Serve frontend estático em /app
   - Rotas públicas: POST /api/auth/register, POST /api/auth/login
   - Rotas protegidas (JWT): GET /api/dashboard, CRUD /api/leads, 
     POST /api/leads/:id/interactions, GET/POST /api/leads/export|import

4. config/firebase.go — Inicialização do Firebase:
   - Coleções isoladas com prefixo:
     * leadmanager_users
     * leadmanager_leads
   - Lê credenciais do caminho configurado no .env

5. middleware/auth.go — Middleware JWT:
   - Valida header "Authorization: Bearer <token>"
   - Injeta userID, userEmail, userName no contexto Gin

6. models/user.go — Model de usuário:
   - Struct User com ID, Name, Email, PasswordHash, CreatedAt
   - RegisterRequest, LoginRequest, AuthResponse

7. models/lead.go — Model de lead:
   - Constantes: StatusNovo, StatusEmContato, StatusQualificado, StatusPerdido
   - Constantes de fonte: indicacao, site, evento, linkedin, email, outro
   - Struct Lead com todos os campos + slice de Interaction
   - Struct Interaction (tipo, nota, criado por, timestamp)
   - CreateLeadRequest, UpdateLeadRequest, AddInteractionRequest
   - LeadListParams (filtros + paginação + ordenação)
   - LeadListResponse, DashboardStats

8. handlers/auth.go — Register e Login:
   - Register: verifica e-mail duplicado, hash bcrypt, salva no Firestore, retorna JWT
   - Login: busca por e-mail, compara hash, retorna JWT

9. handlers/leads.go — CRUD completo:
   - ListLeads: filtro por status/fonte/data/busca livre, ordenação, paginação
   - CreateLead, GetLead, UpdateLead (firestore.Update), DeleteLead
   - AddInteraction: prepend no array de interações
   - ExportLeads: CSV (com BOM UTF-8) ou Excel (.xlsx)
   - ImportLeads: CSV ou Excel, salva em batch no Firestore

10. handlers/dashboard.go — Estatísticas:
    - Total de leads, leads por status, leads por fonte
    - Capturas por dia (últimos 30 dias)
    - 5 leads mais recentes
    - Taxa de conversão (qualificados / total * 100)

11. utils/jwt.go — JWT HS256:
    - GenerateJWT (validade 24h, claims: userID, email, name)
    - ValidateJWT

12. utils/export.go — CSV e Excel:
    - LeadsToCSV: encoding/csv com cabeçalho em português
    - LeadsToExcel: excelize com estilo (cabeçalho azul escuro, larguras definidas)
    - ParseCSVToLeads: parser tolerante (múltiplos formatos de data, 
      normalização de status e fonte)
    - NormalizeStatus, NormalizeSource, StatusLabel

13. frontend/js/api.js — Client HTTP compartilhado:
    - Objeto `api` com todos os métodos (auth, dashboard, leads, export, import)
    - Guard requireAuth() e getUser()
    - Helpers: statusBadge, sourceBadge, formatDate, formatDatetime, initials
    - showToast com tipos success/error/info
    - injectSharedStyles() com estilos de badges e toast

14. frontend/index.html — Login/Cadastro:
    - Layout split: painel esquerdo com grid, hero e stats; painel direito com form
    - Tabs para alternar entre Login e Cadastro
    - Redirect automático se já logado
    - Enter key para submeter

15. frontend/dashboard.html — Dashboard:
    - Sidebar com navegação e info do usuário
    - 4 cards de KPI (Total, Novos, Qualificados, Perdidos)
    - Gráfico de linha: capturas por dia / 30 dias (Chart.js)
    - Gráfico donut: leads por status
    - Lista de fontes com percentuais
    - Tabela de 5 leads recentes com link para detalhe

16. frontend/leads.html — Listagem de Leads:
    - Sidebar idêntica
    - Barra de filtros: busca livre, status, fonte, período (dateFrom/dateTo)
    - Tabela com colunas ordenáveis (nome, empresa, fonte, status, data captura)
    - Ações por linha: ver detalhe, editar, deletar
    - Modal de criação/edição com todos os campos
    - Paginação com navegação por páginas
    - Botões de exportação CSV/Excel
    - Modal de importação com drag-and-drop

17. frontend/lead-detail.html — Detalhe do Lead:
    - Breadcrumb de navegação
    - Hero card com avatar, nome, cargo/empresa, badges de status e fonte
    - Grid de informações (e-mail linkado, telefone, datas)
    - Toggle para modo de edição inline (formulário completo)
    - Painel direito: histórico de interações
      * Seletor de tipo: Nota, Ligação, E-mail, Reunião
      * Textarea com Ctrl+Enter para submeter
      * Timeline de interações em ordem decrescente
    - Modal de confirmação para exclusão

18. README.md — Documentação completa:
    - Estrutura do projeto
    - Pré-requisitos e instalação passo a passo no Windows
    - Todos os endpoints documentados com parâmetros
    - Formato do CSV para importação
    - Solução de problemas comuns
