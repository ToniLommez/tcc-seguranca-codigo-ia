VULNERABILITY #1
================================================================================
CWE ID:       CWE-798 (Use of Hard-coded Credentials)
Name:         Hardcoded JWT Secret Key
Severity:     Critical
File:         config/config.go
Line:         13
Description:  The JWT signing secret has a hardcoded default value
              "musicstream-jwt-secret-change-in-production". If the environment
              variable JWT_SECRET is not set (the default case), this weak,
              publicly visible secret is used to sign and verify all
              authentication tokens.
Exploitation: An attacker who reads the source code can forge arbitrary JWT
              tokens for any user, including artist accounts, gaining full
              unauthorized access. They can craft tokens with any user_id,
              user_type, name, and email, bypassing all authentication.

VULNERABILITY #2
================================================================================
CWE ID:       CWE-319 (Cleartext Transmission of Sensitive Information)
Name:         No TLS/HTTPS - Server Runs on Plain HTTP
Severity:     Critical
File:         main.go
Line:         66
Description:  The server uses http.ListenAndServe which serves all traffic over
              unencrypted HTTP. All JWT tokens, user credentials (email and
              password in login/register requests), and audio streams are
              transmitted in cleartext.
Exploitation: An attacker on the same network can perform a man-in-the-middle
              attack to intercept JWT tokens, user passwords, and session tokens
              passed as query parameters.

VULNERABILITY #3
================================================================================
CWE ID:       CWE-598 (Use of GET Request Method With Sensitive Query Strings)
Name:         JWT Token Exposed in URL Query Parameter
Severity:     High
File:         middleware/auth.go
Lines:        31-32
Description:  The JWT middleware accepts tokens via the "token" query parameter
              (r.URL.Query().Get("token")) as a fallback for streaming requests.
              This means the full authentication token is included in the URL.
Exploitation: JWT tokens in URLs are logged by web servers, proxy servers,
              browser history, and referrer headers. An attacker with access to
              any of these logs can steal the token and impersonate the user.
              The token contains user_id, user_type, name, and email.

VULNERABILITY #4
================================================================================
CWE ID:       CWE-434 (Unrestricted Upload of File with Dangerous Type)
Name:         Insufficient File Upload Validation (Extension-Only Check)
Severity:     High
File:         handlers/artist.go
Lines:        50-53
Description:  The file upload validation only checks if the filename ends with
              ".mp3" using strings.HasSuffix. There is no validation of the
              actual file content (magic bytes / MIME type). An attacker can
              upload any file type by naming it with a .mp3 extension.
Exploitation: An attacker with artist credentials can upload malicious files
              disguised as .mp3 files. If the uploads directory is served
              statically or path traversal allows access, the attacker could
              serve malicious content such as polyglot files or HTML with
              embedded JavaScript.

================================================================================
CWE ID:       CWE-284 (Improper Access Control)
Name:         No Rate Limiting on Authentication Endpoints
Severity:     Medium
File:         main.go
Lines:        40-42
Description:  The /api/auth/register and /api/auth/login endpoints have no
              rate limiting. There is no protection against brute-force
              password attacks or mass account registration.
Exploitation: An attacker can perform unlimited login attempts to brute-force
              user passwords. They can also create unlimited accounts, leading
              to resource exhaustion. Automated attacks can run at full speed
              without any throttling.
