VULNERABILITY #1
--------------------------------------------------------------------------------
CWE ID:        CWE-798 (Use of Hard-Coded Credentials)
Name:          Hardcoded JWT Secret Key
Severity:      CRITICAL
File:          backend/config.py
Line:          4
Code:          SECRET_KEY = os.getenv("SECRET_KEY", "claude-python-leads-secret-2024-change-in-production")

Description:
The JWT signing secret key has a hardcoded default fallback value
"claude-python-leads-secret-2024-change-in-production". If the SECRET_KEY
environment variable is not set, the application will use this predictable,
publicly visible secret to sign and verify all authentication tokens.

Exploitation:
An attacker who reads this source code can forge valid JWT tokens for any user.
This allows complete authentication bypass -- the attacker creates a JWT payload
with the desired user identity, signs it with the known secret, and sends it in
API requests, impersonating any user including administrators.

VULNERABILITY #2
--------------------------------------------------------------------------------
CWE ID:        CWE-942 (Overly Permissive Cross-domain Whitelist)
Name:          Unrestricted CORS Policy (allow_origins=["*"])
Severity:      HIGH
File:          backend/main.py
Lines:         14-20
Description:
The CORS middleware is configured to allow requests from ANY origin ("*"),
with ALL methods and ALL headers permitted, and credentials are allowed. While
browsers may block the combination of allow_origins=["*"] with
allow_credentials=True, many CORS implementations reflect the Origin header
when "*" is used, effectively allowing credentialed cross-origin requests from
any domain.

Exploitation:
An attacker can host a malicious website that makes cross-origin requests to
this API using the victim's credentials. If a logged-in user visits the
attacker's page, JavaScript on that page can make authenticated API requests to
the Lead Management system, reading sensitive lead data, creating or modifying
leads, or performing any action the victim is authorized to do.

--------------------------------------------------------------------------------
VULNERABILITY #3
--------------------------------------------------------------------------------
CWE ID:        CWE-862 (Missing Authorization)
Name:          No Centralized Authorization Framework
Severity:      HIGH
File:          backend/main.py
Lines:         12-23
Description:
The main application file sets up routers for authentication and leads but does
not configure any authorization middleware, role-based access control, or
resource ownership verification at the application level. Each endpoint must
independently implement access checks, which is error-prone.

Exploitation:
If any route handler fails to implement its own authorization check, the
endpoint becomes accessible to any authenticated user (or even unauthenticated
users). Without centralized authorization, a single developer oversight exposes
the endpoint. Common consequences include IDOR (Insecure Direct Object
Reference) where one user can access or modify another user's leads.
