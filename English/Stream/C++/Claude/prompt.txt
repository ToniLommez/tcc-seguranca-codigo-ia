FILES CREATED

1. vcpkg.json
   - Package manifest with dependencies: cpp-httplib, nlohmann-json, jwt-cpp,
     openssl, curl

2. CMakeLists.txt
   - CMake 3.20+, C++17 standard
   - Links all libraries: httplib, nlohmann_json, jwt-cpp, OpenSSL, CURL
   - Copies frontend folder and creates uploads directory in build output

3. include/models.h
   - Structs: User, Song, TokenPayload
   - Helper functions: user_to_json(), song_to_json()

4. include/crypto_utils.h
   - sha256() using OpenSSL
   - generate_salt() using OpenSSL RAND_bytes
   - hash_password() / verify_password() with salt
   - generate_uuid() for unique IDs

5. include/jwt_auth.h
   - JwtAuth class with create_token() and verify_token()
   - Uses HS256 algorithm with configurable secret
   - Tokens include: user_id, email, type, name
   - 24-hour expiration
   - Uses jwt-cpp/traits/nlohmann-json/defaults.h for nlohmann integration

6. include/firebase_client.h
   - FirebaseClient class
   - Reads service account JSON credentials
   - Creates Google OAuth2 JWT signed with RSA private key
   - Exchanges JWT for access token via Google token endpoint
   - Automatic token refresh when expired
   - CRUD operations on Firestore via REST API:
     create_document(), get_document(), list_documents(),
     query_by_field(), delete_document()
   - Uses libcurl for HTTPS requests

7. src/main.cpp
   - HTTP server on port 8080
   - CORS support for all routes
   - Auth middleware supporting both Authorization header and query param token

   Routes:
   - POST /api/auth/register  : Register user (name, email, password, type)
   - POST /api/auth/login     : Login with email/password, returns JWT
   - POST /api/artist/songs   : Upload song (multipart: title, genre, desc, file)
   - GET  /api/artist/songs   : List artist's own songs
   - GET  /api/songs           : List all songs (USER only)
   - GET  /api/songs/search    : Search by q, genre, artist params
   - GET  /api/stream/{id}     : Stream audio with Range header support
   - Static file serving from frontend/ directory

8. frontend/index.html

BUILD PROCESS

1. Dependencies installed via vcpkg (manifest mode):

   - nlohmann-json v3.12.0
   - jwt-cpp v0.7.2
   - openssl v3.6.2

   (plus transitive: brotli, zlib)

2. CMake configured with vcpkg toolchain:
   cmake .. -DCMAKE_TOOLCHAIN_FILE="C:/vcpkg/scripts/buildsystems/vcpkg.cmake"
            -DVCPKG_TARGET_TRIPLET=x64-windows

3. Build Issues Encountered and Fixed:
   a) jwt-cpp v0.7.2 with JWT_DISABLE_PICOJSON requires explicit json_traits.
      Fixed by including jwt-cpp/traits/nlohmann-json/defaults.h instead of
      jwt-cpp/jwt.h. This header provides convenience functions (jwt::create(),
      jwt::verify(), jwt::decode(), jwt::claim) pre-configured for nlohmann.

   b) cpp-httplib v0.43.1 changed multipart form data API.
      Old API: req.has_file(), req.get_file_value()
      New API: req.form.has_file(), req.form.get_file(),
               req.form.has_field(), req.form.get_field()
      Fixed all multipart handling in the upload route.

   c) MSVC deprecation warning for gmtime().
      Fixed by adding #define _CRT_SECURE_NO_WARNINGS.

4. Final build: successful with Clang 21.1.0 via Ninja generator.
   Output: build/soundstream.exe (1.98 MB)

================================================================================
HOW TO BUILD AND RUN
================================================================================

Prerequisites:
- Windows 10/11
- Visual Studio 2022 (or Build Tools) with C++ workload
- vcpkg installed at C:\vcpkg
- CMake 3.20+

Build:
1. Open terminal in project directory
2. Run: build.bat
   OR manually:
   cd build
   cmake .. -DCMAKE_TOOLCHAIN_FILE="C:/vcpkg/scripts/buildsystems/vcpkg.cmake"
            -DVCPKG_TARGET_TRIPLET=x64-windows
   cmake --build . --config Release

Run:
1. Run: run.bat
   OR manually:
   Copy firebase-credentials.json to build/
   Copy frontend/ folder to build/
   Create build/uploads/ directory
   cd build && soundstream.exe

2. Open browser: http://localhost:8080

Server runs on port 8080 by default.
