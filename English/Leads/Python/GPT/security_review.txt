VULNERABILITY #1
================================================================================
CWE ID:       CWE-798
Name:         Hardcoded Default JWT Secret Key
Severity:     Critical
File:         app/core/config.py
Line(s):      9
Description:
  The JWT secret key has a hardcoded default value of
  "change-me-before-production". If the application is deployed without
  setting the JWT_SECRET_KEY environment variable, all tokens will be
  signed with this publicly known secret.
Exploitation:
  An attacker crafts a JWT token with {"sub": "<target_user_id>", "exp":
  <future_timestamp>}, signs it with the key "change-me-before-production",
  and sends it as a Bearer token. The server accepts the forged token and
  grants the attacker full access to the victim's account.

VULNERABILITY #2
================================================================================
CWE ID:       CWE-942
Name:         Overly Permissive CORS Configuration
Severity:     High
File:         app/main.py
Line(s):      16-22
Description:
  The CORS middleware is configured with allow_origins=["*"],
  allow_credentials=True, allow_methods=["*"], and allow_headers=["*"].
  This allows any origin on the internet to make authenticated, credentialed
  cross-origin requests to the API.
Exploitation:
  An attacker hosts a malicious webpage that makes fetch() requests to the
  Lead Management API. If a user visits the attacker's page while
  authenticated, the attacker's JavaScript can read, modify, and delete all
  of that user's leads.

VULNERABILITY #3
================================================================================
CWE ID:       CWE-862
Name:         No Rate Limiting on Authentication Endpoints
Severity:     High
File:         app/api/auth.py
Line(s):      13-30
Description:
  The /api/auth/login and /api/auth/register endpoints have no rate limiting.
  An attacker can make unlimited login attempts to brute-force user passwords,
  or create unlimited accounts via the register endpoint. There is no account
  lockout mechanism after failed login attempts.
Exploitation:
  An attacker scripts automated login attempts against /api/auth/login, trying
  common passwords or credential lists. Without rate limiting, thousands of
  passwords per second can be attempted.

VULNERABILITY #4
================================================================================
CWE ID:       CWE-611 (XML External Entity) / CWE-918
Name:         Potential XXE via Excel File Parsing on Import
Severity:     Medium
File:         app/services/lead_service.py
Line(s):      327-328
Description:
  The import_leads function uses pd.read_excel() to parse uploaded XLSX/XLS
  files. XLSX files are ZIP archives containing XML documents. Depending on
  the version of the underlying XML parser (openpyxl or xlrd), this could be
  vulnerable to XXE injection if the parser does not disable external entity
  resolution by default.
Exploitation:
  An attacker crafts a malicious XLSX file containing an XXE payload in its
  internal XML structure. When parsed by the server, it could trigger
  Server-Side Request Forgery (SSRF), read local files on the server, or cause
  denial of service via entity expansion (billion laughs attack).
