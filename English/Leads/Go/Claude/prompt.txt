Requirements
Authentication and User Module
The system should allow new users (companies/managers) to sign up with name,
email, and password.
Users should be able to log in using their email and password, receiving a
valid JWT token upon authentication.

Lead Management Module
The system should allow creating leads with the following fields: full name,
email, phone, company, position, source (referral, website, event, etc.),
status (new, contacted, qualified, lost), and capture date.
Full CRUD operations for leads must be supported.
The system should list all leads with pagination and sorting.
It should be possible to filter leads by status, source, capture date, and
free text (name, email, company).
The system should allow viewing complete details for a specific lead.
Batch import and export of leads via CSV/Excel files should be supported.

Frontend Module
The frontend should include a user login/registration screen.
It should have a dashboard showing lead summaries by status, captures over
time, and similar metrics.
It should provide forms for creating and editing leads.
It should include a lead details page with interaction history.
The frontend should be designed with a desktop-first approach.

Built a full-stack lead management system in Go with the following components:

PROJECT STRUCTURE:
  cmd/server/main.go                  - Server entry point, HTTP routing
  internal/auth/jwt.go                - JWT token generation & validation
  internal/firebase/client.go         - Firebase/Firestore singleton client
  internal/models/user.go             - User, SignupRequest, LoginRequest, AuthResponse structs
  internal/models/lead.go             - Lead, Interaction, LeadFilter, PaginatedResponse structs
  internal/handlers/auth.go           - Signup & login API handlers
  internal/handlers/leads.go          - Full CRUD + stats + interactions handlers
  internal/handlers/import_export.go  - CSV/Excel import & export handlers
  internal/csvutil/csvutil.go         - CSV/Excel parsing & generation logic
  internal/middleware/auth.go         - JWT Bearer auth middleware + CORS middleware
  web/static/css/style.css            - Full desktop-first CSS (sidebar layout, modals, charts)
  web/static/js/api.js                - Frontend API client (fetch wrapper + auth helpers)
  web/static/js/utils.js              - Shared utilities (toast, date format, debounce, etc.)
  web/templates/index.html            - Login / signup screen
  web/templates/dashboard.html        - Stats dashboard with bar chart and donut chart
  web/templates/leads.html            - Paginated leads table with filters, sort, import/export
  web/templates/lead-detail.html      - Lead detail view + interaction history
  run.bat                             - Windows startup script
