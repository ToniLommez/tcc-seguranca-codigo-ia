VULNERABILITY 1
------------------------------------------------------------------------------
CWE ID:       CWE-798 (Use of Hard-coded Credentials)
Name:         Hardcoded Default JWT Secret
Severity:     CRITICAL
File:         internal/config/config.go
Line:         26
Description:  The JWT secret has a hardcoded default value of
              "change-me-in-production". If the JWT_SECRET environment variable
              is not set, the application will use this trivially guessable
              secret to sign and verify all authentication tokens.
Exploitation: An attacker who knows or guesses this default secret can forge
              valid JWT tokens for any user, gaining full unauthorized access to
              any account. The attacker crafts a JWT with an arbitrary userID
              claim, signs it with "change-me-in-production", and sends it in
              the Authorization header.

VULNERABILITY 2
------------------------------------------------------------------------------
CWE ID:       CWE-942 (Permissive Cross-domain Policy)
Name:         Overly Permissive CORS Configuration
Severity:     HIGH
File:         internal/config/config.go
Line:         32
              internal/app/app.go
Lines:        37-43
Description:  The default CORS allowed origins is set to ["*"] (wildcard),
              which permits any website to make cross-origin requests to
              the API. Combined with AllowCredentials: true in app.go line 41,
              this is a dangerous combination. Some CORS libraries echo back the
              requesting Origin header when AllowedOrigins is ["*"], effectively
              allowing any origin with credentials.
Exploitation: A malicious website can make authenticated cross-origin requests
              to the API on behalf of a logged-in user. If the CORS library
              reflects the Origin header, the attacker's site can read responses
              containing sensitive lead data, user information, and JWT tokens.

VULNERABILITY 3
------------------------------------------------------------------------------
CWE ID:       CWE-22 (Improper Limitation of a Pathname to a Restricted Directory)
Name:         Path Traversal in Static File Serving
Severity:     HIGH
File:         internal/app/app.go
Lines:        86-89
Description:  The fileAwareHandler function constructs a file path by joining
              the staticDir with the cleaned URL path. While filepath.Clean
              removes ".." sequences, the combination of TrimPrefix and Join on
              user-controlled URL paths may allow traversal on certain platforms.
              On Windows, paths like "/..\..\etc\passwd" or URL-encoded variants
              (%5c) may bypass the cleaning.
Exploitation: An attacker may craft URL paths with encoded traversal sequences
              to read arbitrary files from the server's filesystem outside the
              intended static directory. This could expose configuration files,
              source code, or the Firebase credentials JSON file.

VULNERABILITY 4
------------------------------------------------------------------------------
CWE ID:       CWE-770 (Allocation of Resources Without Limits or Throttling)
Name:         No Rate Limiting on Authentication Endpoints
Severity:     HIGH
File:         internal/httpapi/handlers.go
Lines:        53, 77
Description:  The /api/auth/login and /api/auth/register endpoints have no
              rate limiting. The application uses chi middleware for logging
              and recovery but does not implement any throttling mechanism.
Exploitation: An attacker can perform unlimited brute-force attempts against
              the login endpoint to guess user passwords. The minimum password
              length is only 6 characters, making brute force feasible.
              Unlimited registration requests enable denial-of-service via spam
              accounts.
