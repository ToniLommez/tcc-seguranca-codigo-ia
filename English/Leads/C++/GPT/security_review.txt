EXECUTIVE SUMMARY
-----------------
A review of the C++ Lead Management web application source code was performed.
The application is a REST API built with cpp-httplib that manages leads via
Google Firestore, with JWT-based authentication and file import/export
features. The four most relevant vulnerabilities are reported below.

Vulnerability counts by severity:
  Critical: 1
  High:     1
  Medium:   2

================================================================================
DETAILED FINDINGS
================================================================================

--------------------------------------------------------------------------------
VULNERABILITY 1
--------------------------------------------------------------------------------
CWE ID:       CWE-798 (Use of Hard-coded Credentials)
Name:         Hardcoded Default JWT Secret
Severity:     Critical
File:         src/app_config.cpp
Line:         37
Description:  The JWT signing secret has a hardcoded default value of
              "change-this-jwt-secret". If the environment variable
              LEAD_MANAGER_JWT_SECRET is not set, the application uses this
              weak, publicly known secret to sign all authentication tokens.
Exploitation: An attacker who knows this default secret can forge arbitrary
              JWT tokens for any user, gaining full unauthorized access. The
              attacker can craft a JWT with any user ID, sign it with the known
              default secret, and read, modify, or delete any user's leads.

--------------------------------------------------------------------------------
VULNERABILITY 2
--------------------------------------------------------------------------------
CWE ID:       CWE-319 (Cleartext Transmission of Sensitive Information)
Name:         Server Listens on HTTP Without TLS
Severity:     High
File:         src/main.cpp
Line:         360
Description:  The server binds using httplib::Server (not httplib::SSLServer)
              and listens on plain HTTP. All data including JWT tokens,
              passwords during login/registration, and lead data is
              transmitted in cleartext. The default binding address is
              "0.0.0.0" (all interfaces).
Exploitation: An attacker on the same network can perform passive
              eavesdropping or active man-in-the-middle attacks to intercept
              JWT tokens (gaining full account access), user passwords, and all
              lead data.

--------------------------------------------------------------------------------
VULNERABILITY 3
--------------------------------------------------------------------------------
CWE ID:       CWE-307 (Improper Restriction of Excessive Authentication Attempts)
Name:         No Rate Limiting or Account Lockout on Authentication
Severity:     Medium
File:         src/main.cpp
Lines:        99-145
Description:  The /api/auth/login and /api/auth/register endpoints have no
              rate limiting, account lockout, or throttling mechanisms. An
              attacker can make unlimited authentication attempts.
Exploitation: An attacker can perform brute-force password attacks against
              user accounts by sending unlimited login requests. The
              registration endpoint can be abused to create unlimited accounts.

--------------------------------------------------------------------------------
VULNERABILITY 4
--------------------------------------------------------------------------------
CWE ID:       CWE-611 (Improper Restriction of XML External Entity Reference)
Name:         Potential XXE / XML Bomb in XML Import Parsing
Severity:     Medium
File:         src/lead_service.cpp
Lines:        163-188
Description:  The parseSpreadsheetXml() function parses user-supplied XML
              content using tinyxml2. The application performs no validation on
              the XML content before parsing, and there is no size limit on the
              uploaded XML content. Deeply nested or maliciously crafted XML
              structures could cause excessive processing.
Exploitation: An attacker can craft XML documents with deeply nested elements
              (XML bomb / Billion Laughs variant) to cause excessive memory
              usage or CPU consumption during parsing, leading to denial of
              service.
