--- VULNERABILITY 1 ---
CWE ID: CWE-798 (Use of Hard-coded Credentials)
Name: Hardcoded JWT Secret Key
Severity: Critical
File: backend/app/core/settings.py
Line: 9
Description:
  The JWT secret key is hardcoded as a default value:
  jwt_secret_key: str = "change-this-secret-key"
  If no .env file is provided or the environment variable is not set, the
  application runs with this trivially guessable secret.
Exploitation:
  An attacker who discovers (or guesses) the default secret can forge
  arbitrary JWT tokens, impersonate any user (including artists), upload
  malicious files, and access all protected endpoints.

--- VULNERABILITY 2 ---
CWE ID: CWE-942 (Overly Permissive Cross-Origin Resource Sharing)
Name: Wildcard CORS Configuration with Credentials
Severity: High
File: backend/app/main.py
Lines: 22-28
Description:
  The CORS middleware is configured with allow_origins=["*"],
  allow_credentials=True, allow_methods=["*"], and allow_headers=["*"]. The
  combination of allow_origins=["*"] with allow_credentials=True is
  particularly dangerous.
Exploitation:
  An attacker can host a malicious website that makes authenticated requests
  to the API on behalf of any logged-in user, enabling cross-site request
  forgery attacks, data theft, and unauthorized actions such as uploading
  songs from any domain.

--- VULNERABILITY 3 ---
CWE ID: CWE-319 (Cleartext Transmission of Sensitive Information)
Name: JWT Token Accepted via Query Parameter
Severity: High
File: backend/app/api/dependencies.py
Lines: 17-21
Description:
  The get_current_user function accepts the JWT access token via a query
  parameter (access_token: str | None = Query(default=None)) as a fallback
  when no Authorization header is provided. This means authentication tokens
  can be passed in the URL.
Exploitation:
  JWT tokens passed as query parameters are logged in web server access logs,
  browser history, proxy logs, Referer headers, and any URL logging
  middleware. An attacker who gains access to any of these log sources obtains
  valid session tokens and can impersonate users.

--- VULNERABILITY 4 ---
CWE ID: CWE-434 (Unrestricted Upload of File with Dangerous Type)
Name: Insufficient File Upload Validation
Severity: High
File: backend/app/api/artist.py
Lines: 60-65
Description:
  The song upload endpoint only validates the file extension
  (extension = Path(file.filename or "").suffix.lower(); if extension != ".mp3").
  This check relies solely on the client-provided filename. There is no
  validation of the file's actual content/MIME type (magic bytes), no file size
  limit, and no Content-Type header validation.
Exploitation:
  An attacker (authenticated as an artist) can upload a malicious file (e.g.,
  executable, HTML with JavaScript) renamed with a .mp3 extension. Since
  file_path is stored as an absolute path and the uploads directory is
  accessible, this could lead to stored XSS or server compromise if the file is
  served with the wrong content type or later executed.
