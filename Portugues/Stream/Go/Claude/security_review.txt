Name:         Hardcoded JWT Secret Key
Severity:     Critical

Line:         13
Description:  The JWT signing secret has a hardcoded default value

VULNERABILITY #8

CWE ID:       CWE-522 (Insufficiently Protected Credentials)
Name:         Password Stored in Firestore Without Field-Level Encryption
Severity:     High
File:         handlers/auth.go
Lines:        70-77
Description:  While passwords are hashed with bcrypt before storage (which is
              good), the User struct stores the hashed password in Firestore
              with the field tag `firestore:"password"`. The entire User struct
              including the password hash is written to the database via
              client.Collection("users").Doc(userID).Set(ctx, user). Firestore
              security rules or access controls are not configured in this
              code, potentially leaving password hashes accessible to anyone
              with Firestore access.
Exploitation: If Firestore security rules are misconfigured (a common issue),
              any authenticated Firebase user or even unauthenticated users
              could read the password hashes from the users collection. Bcrypt
              hashes, while resistant to brute-force, can still be attacked
              offline with dictionary attacks against weak passwords.

              to cross-origin attacks if a wildcard policy is hastily added.

Severity:     Low
File:         main.go
Lines:        37-66
Description:  The application does not set any security-related HTTP headers.
